# Описание проекта

Давай создадим веб-приложение на FastAPI. Данное приложение представляет собой небольшой SaaS проект по предоставлению доступа к LLM.

Вот примерно как я его вижу:

- Пользователь авторизуется на сайте. Авторизация может происходить как через Email, так и через Google и Github.
- Каждый день пользователю даётся квота в 100 кредитов.
- Пользователь тратит эти кредиты на взаимодействие (общение) с LLM. Доступ к LLM предоставляется как через веб-интерфейс, так и через API.
- Общаться с LLM пользователь может в разных чатах (как в chatgpt или arena.ai).
- Стоимость генерации зависит от модели генерации, количества входных и выходных токенов (как это обычно делается у других провайдеров).
- В сообщении помимо текста польльзователь может прикреплять разные файлы, изображения, документы, аудио, голосовые сообщения.
- В качестве ответа пользователь помимо текста может получить файлы, изображения, документы, аудио и т.д.
- Модели LLM должны указываться в специальной папке. Это могут быть как простые модели, так и LLM агенты. Сам код LLM указывается разработчиком.
